🔍 Flujo 3 · Verificación de Precio por Venta (get_precio_tipo)

flowchart LR classDef input fill:#fef3c7,stroke:#f59e0b,color:#78350f,rx:8 classDef fn fill:#ede9fe,stroke:#7c3aed,color:#3b0764,rx:8 classDef db fill:#dbeafe,stroke:#2563eb,color:#1e3a8a,rx:8 classDef hist fill:#fce7f3,stroke:#db2777,color:#831843,rx:8 classDef out fill:#dcfce7,stroke:#16a34a,color:#14532d,rx:8 ITEM["sales_invoice_items\nprecio_unitario: $4.05\nerp_product_id: 7\nid_presentacion: 12\ninvoice_id: → fecha: 2026-03-15"]:::input ITEM --> RPC["get_precio_tipo(\n precio_unitario,\n product_id,\n id_presentacion,\n fecha\n)"]:::fn RPC --> HIST_Q["Busca en product_precios_history\nDONDE product_id = 7\nAND id_presentacion = 12\nAND valid_from ≤ fecha\nAND (valid_until IS NULL OR valid_until > fecha)\nORDER BY valid_from DESC LIMIT 1"]:::hist HIST_Q --> COMPARE["Compara precio_unitario\nvs cada tier (±$0.005):\nvineta=$4.75 ✗\ndescuento_1=? ✗\nvip=$4.05 ✓\nclinica=$3.90 ✗\nmayoreo=$3.80 ✗\npremium=? ✗\nprecio_7=? ✗"]:::fn COMPARE --> RESULT["→ 'vip'"]:::out ITEM --> ITEMS_DB[("sales_invoice_items")]:::db ITEMS_DB -.->|"erp_product_id + id_presentacion"| HIST_DB[("product_precios_history")]:::hist HIST_DB -.->|"histórico vigente en fecha"| COMPARE
¿Qué dice esto? Dado un ítem de venta con precio $4.05, la función busca los precios que tenía ese producto en esa presentación el día de la venta, y determina que se vendió al tier VIP. Si el precio hubiera cambiado en otro momento, el historial lo captura para que la comparación siempre use el precio correcto de esa fecha.
